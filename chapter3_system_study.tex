\chapter{System Study}
\label{ch:systemstudy}

The system study phase involves understanding existing diagram digitization challenges and analysing requirements for the Automated Diagram Digitization system, covering problem definition, feasibility analysis, and requirements gathering.

%---------------------------------------------------------------
\section{Existing System: Challenges and Disadvantages}
\label{sec:existing-system}
%---------------------------------------------------------------
 
The existing process of digitizing flowcharts faces several critical challenges. Manual redrawing is almost always required, with users forced to recreate diagrams in tools such as Microsoft Visio or Lucidchart—a process taking up to 30 minutes for moderately complex flowcharts while introducing transcription errors. Current systems provide no automated shape recognition, meaning each element must be identified and entered individually. Standard OCR tools fail to preserve text formatting and line breaks, resulting in merged text requiring extensive manual correction. A significant connection detection gap exists, as existing systems cannot recognize directional relationships. Most tools are further limited to manual drag-and-drop input, lack intelligent editing features such as auto-layout and undo/redo, offer restricted export options, and provide no theme or accessibility support.

%---------------------------------------------------------------
\section{Proposed System}
\label{sec:proposed-system}
%---------------------------------------------------------------
 
The Automated Diagram Digitization system is a web-based platform combining AI-powered recognition with an interactive diagram editor for a complete workflow from printed images to editable digital diagrams. The AI recognition module leverages Google Gemini with custom prompts to identify shapes, extract embedded text, and determine connection paths. The platform supports image uploads, natural language descriptions for algorithm-to-flowchart conversion, and direct JSON import. The interactive editor provides drag-and-drop from a shape palette, snapshot-based undo/redo, node resizing, auto-layout via the Dagre engine, edge labelling, text and colour styling, and persistent storage in PostgreSQL with UUID-based identification. Export to PNG, SVG, and JSON is supported, along with dark/light theme switching.
 
The proposed system directly addresses existing limitations through automatic shape identification, text extraction with line-break preservation, automatic directional connection detection, multiple input methods, comprehensive editing features, diverse export options, and persistent cross-session storage.

\section{Feasibility Study}
\label{sec:feasibility}
%---------------------------------------------------------------

A thorough feasibility study was conducted to evaluate whether the 
proposed system can be successfully developed and deployed within the 
available resources. The study examines technical, operational, and 
economic dimensions to confirm that the project is both achievable 
and sustainable.

\subsection{Technical Feasibility}
\label{sec:tech-feasibility}

The project utilises proven, mature technologies that have demonstrated reliability across numerous deployments. The backend is powered by FastAPI which provides high-performance asynchronous endpoints and the Google Gemini API offers state-of-the-art multimodal capabilities for image analysis and text generation. React 19 with Vite and React Flow provide a specialised frontend, and PostgreSQL with JSONB ensures ACID-compliant, flexible data persistence. Finally, the system’s modular architecture supports cloud deployment with horizontal scaling capabilities as user numbers grow.

\subsection{Operational Feasibility}
\label{sec:op-feasibility}

Beyond technical considerations, the system is designed to be practically usable by the intended audience. The platform features a user-friendly interface with a tabbed layout for image upload, text prompts, and JSON import. Interaction within the editor uses familiar conventions such as drag-and-drop, keyboard shortcuts (\texttt{Ctrl+Z}/\texttt{Ctrl+Y}), and double-click edge labelling with clear visual feedback on progress, status, and errors. Responsive design principles ensure that the interface remains usable across a wide range of devices, including desktops, tablets, and mobile phones.

\subsection{Economic Feasibility}
\label{sec:economic-feasibility}

Open-source technologies (FastAPI, React, PostgreSQL) minimise initial investment. The Google Gemini API offers competitive pricing with free-tier options suitable for the development phase. Automation of diagram digitization directly reduces labour costs, and a scalable cloud architecture avoids large upfront infrastructure expenses.

\section{Requirements Analysis}
\label{sec:requirements}
%---------------------------------------------------------------

Requirements analysis is a critical phase that bridges the gap 
between problem identification and system design. It involves 
identifying, documenting, and organizing the specific capabilities 
and constraints that the system must fulfill.

\subsection{Functional Requirements}
\label{sec:functional-req}

Based on user needs and system objectives, the following functional 
requirements define the core capabilities required for efficient 
diagram digitization and interaction.

The system includes an input module supporting multi-format image 
uploads (PNG, JPG, WebP) with drag-and-drop and file size validation, 
a multi-line text prompt with character counting for algorithm-to-flowchart 
conversion, and direct JSON import with validation. The AI processing 
module identifies flowchart elements such as process, decision, start, 
and input/output shapes, extracts embedded text with multi-line 
preservation, detects directional edges with appropriate connection 
handles, and incorporates a regex-based fallback to handle variability 
in AI-generated responses.

The editor module enables interactive manipulation through drag-and-drop 
from a shape palette, node resizing with adaptive text, snapshot-based 
undo/redo with keyboard shortcuts, and a column-based auto-layout 
mechanism for structured positioning. It also supports edge labeling, 
automatic snapshot capture on deletion, and styling options for nodes, 
edges, and text. The export module provides PNG generation via 
\texttt{html-to-image}, SVG export through React Flow, and JSON 
serialization for portability. Additionally, the storage module ensures 
persistent data management using PostgreSQL with UUID-based identification, 
timestamp tracking, and support for retrieval and updates to reconstruct 
and modify the editor state.


\subsection{Non-Functional Requirements}
\label{sec:nonfunctional-req}

Non-functional requirements define the quality attributes of the 
system, describing how effectively it performs its operations in 
terms of performance, usability, reliability, and security, and are 
essential for ensuring a robust user experience and system stability.

The system maintains efficient performance with fast image analysis 
and text-to-flowchart generation, responsive editor interactions, 
and support for multiple concurrent users without noticeable 
degradation. Usability is ensured through an intuitive interface 
requiring minimal training, clear error messages, tooltips, keyboard 
shortcuts, and responsive design with consistent theming across 
devices. Reliability is achieved through stable system operation, 
graceful error handling with fallback mechanisms, and data integrity 
maintained via transaction support and database backup procedures. 
Security is enforced by protecting the Gemini API key through 
server-side environment variables, sanitizing inputs, configuring 
CORS appropriately, preventing SQL injection using parameterized 
queries, and maintaining separate configurations for development 
and production environments.

\section{System Modules}
\label{sec:system-modules}
%---------------------------------------------------------------

The Automated Diagram Digitization platform is organized into five 
functional modules, each responsible for a specific part of the 
overall workflow. This modular design simplifies development and 
maintenance while allowing individual components to be improved 
independently.

The input module handles image upload, text prompt, and JSON import with validation and pre-processing. The AI processing module interfaces with Gemini AI using custom prompts, parses JSON responses with regex fallback, and ensures proper edge ID generation. The editor module renders nodes and edges via React Flow with custom node components, and implements drag-and-drop, undo/redo, auto-layout, edge labelling, and comprehensive styling. The export module produces PNG via \texttt{html-to-image}, SVG via React Flow's renderer, and JSON of the current flow state. The storage module manages PostgreSQL CRUD operations with UUID identification, timestamp tracking, and JSONB storage.

\section{Conclusion}
\label{sec:ch3-conclusion}
%---------------------------------------------------------------

The system study has established clear and comprehensive requirements 
for the Automated Diagram Digitization project, addressing critical 
gaps in automated shape recognition, text extraction, connection 
detection, and interactive editing. The analysis of existing systems 
revealed significant inefficiencies in manual redrawing workflows, 
the absence of automation, and limited editing capabilities in 
current tools. The proposed system directly addresses these challenges through 
AI-powered recognition, multiple input methods, comprehensive editing 
features, and flexible export options. Feasibility analysis confirms 
that the solution is technically achievable using proven technologies 
such as FastAPI, React, PostgreSQL, and Google Gemini AI, which 
collectively provide a stable and scalable foundation.

From an operational perspective, the system offers a user-friendly 
interface that simplifies the overall workflow from diagram input to 
final export. Economically, the use of open-source technologies and 
cloud-based infrastructure reduces development and operational costs 
while providing long-term sustainability. Chapter~4 presents the 
detailed system design, translating the requirements established here 
into architectural, data flow, and module-level design 
specifications.
