\chapter{Testing}
\label{ch:testing}

The validation of system performance and functional accuracy is a critical phase in the development of the Automated Diagram Digitization platform. This chapter details the comprehensive testing methodology employed to ensure that the platform meets the high standards of reliability and usability required by students and professionals. By systematically executing unit, integration, and user acceptance tests, our team has identified and resolved technical defects, optimized processing speeds, and refined the interactive experience of the flowchart editor.

Testing ensures the reliability, functionality, and quality of the Automated Diagram Digitization system across all modules. Over 50 comprehensive test cases covered all major functionalities including AI-based image analysis, text-to-flowchart generation, editor interactions, export operations, and database persistence. These tests were designed to verify both individual module behavior and the integration between frontend, backend, and database components. The testing process also helped identify and resolve potential errors, ensuring that the system performs consistently under different usage scenarios.~\cite{softwaretesting}

\section{Testing Levels}

The testing strategy for the Automated Diagram Digitization platform is structured around three complementary levels of testing. Each level examines the system from a different perspective, ensuring that individual components, module interactions, and the complete system all function correctly and meet the defined requirements.~\cite{sommerville, pressman}

\subsection{Unit Testing}

Unit testing involves verifying the correctness of individual components in isolation, independent of the rest of the system. By testing each module separately, developers can quickly identify and fix defects at the source before they propagate to other parts of the application. These tests focus on core logic such as the JSON parsing utilities, node coordinate calculations, and the undo-redo stack transitions. Using mocked dependencies ensures that the tests remain deterministic and execute rapidly, providing a tight feedback loop during development. Furthermore, edge cases such as empty input strings and malformed coordinate pairs are systematically validated to prevent runtime exceptions in the production environment. The following components were covered during unit testing.
\begin{enumerate}
    \item \textbf{Input Module}: File upload validation, drag-drop handling, JSON parsing
    \item \textbf{AI Processing}: Image preprocessing, Gemini API communication, JSON response parsing
    \item \textbf{Editor Functions}: Node addition/deletion, edge creation, undo/redo operations
    \item \textbf{Auto-Layout}: Column clustering algorithm, node repositioning calculations
    \item \textbf{Export Functions}: PNG capture, SVG generation, JSON serialization
    \item \textbf{Database}: CRUD operations, UUID generation, timestamp tracking
\end{enumerate}

\subsection{Integration Testing}

Integration testing verifies that different modules of the system work together correctly when combined. While unit testing ensures individual components function properly, integration testing focuses on the data flow and communication between modules. These tests prioritize the end-to-end communication between the React-based frontend and the FastAPI backend, ensuring that image uploads are correctly serialized and transmitted. Special attention is given to the synchronization of state between the React Flow canvas and the database persistence layer during batch update operations. By validating the integrity of cross-module data structures, the system maintains consistent behavior even as individual components evolve. The following key workflows were validated during integration testing.
\begin{enumerate}
    \item Image upload $\rightarrow$ AI analysis $\rightarrow$ Editor rendering $\rightarrow$ User edits
    \item Text prompt $\rightarrow$ Gemini generation $\rightarrow$ JSON parsing $\rightarrow$ Diagram display
    \item JSON import $\rightarrow$ Validation $\rightarrow$ Editor state restoration
    \item Editor edits $\rightarrow$ Undo/Redo $\rightarrow$ History navigation
    \item Auto-layout $\rightarrow$ Node repositioning $\rightarrow$ Edge type adjustment
    \item Save flowchart $\rightarrow$ Database storage $\rightarrow$ Retrieve by ID $\rightarrow$ Re-render
\end{enumerate}

\subsection{System Testing}

System testing evaluates the complete, integrated application as a whole to verify that it meets all specified functional and non-functional requirements. This level of testing simulates real-world usage scenarios and examines the system under conditions similar to those experienced by end users. It also addresses cross-cutting concerns such as responsiveness across different display resolutions and the system's ability to maintain high performance during complex diagram generation tasks. Load testing is conducted to determine the platform's stability under multiple concurrent user requests. Finally, usability assessments are performed to verify that the integrated workflow from initial upload to final export provides a seamless and intuitive experience. The following categories were included in the system-level test plan.
\begin{enumerate}
    \item \textbf{Functional testing}: All features across all modules
    \item \textbf{UI/UX testing}: Responsiveness, intuitiveness, theme switching
    \item \textbf{Security testing}: API key protection, input validation, CORS
    \item \textbf{Performance testing}: Response times and resource usage
    \item \textbf{Cross-browser testing}: Chrome, Firefox, Safari, Edge
    \item \textbf{Responsive design testing}: Desktop, tablet, mobile views
    \item \textbf{Error handling}: Malformed AI responses, invalid inputs, network failures
\end{enumerate}

\section{Security Testing}

Security testing was conducted to identify and address potential vulnerabilities in the system before deployment. Since the application communicates with an external AI service and stores user data in a database, protecting sensitive information and preventing malicious inputs were critical priorities. The following measures were evaluated to confirm that the system maintains a robust security posture. We also simulated various edge cases, such as the injection of malformed JSON payloads and oversized image binaries, to ensure that the backend's error-handling mechanisms respond gracefully without exposing internal system metadata. These rigorous assessments validate that the platform is fortified against common web security threats, providing a safe environment for both personal and collaborative diagramming.
Security testing focused on protecting the Gemini API key and preventing malicious inputs:

\begin{enumerate}
    \item \textbf{API Key Protection}: Key stored in environment variables, never in client code
    \item \textbf{Input Validation}: File type and size validation before processing
    \item \textbf{SQL Injection}: Parameterized queries with async SQLAlchemy
    \item \textbf{CORS Configuration}: Restricted to allowed origins (localhost:5173, localhost:3000)
    \item \textbf{JSON Parsing}: Safe parsing with error handling for malformed data
    \item \textbf{File Upload Security}: Image validation using Pillow, size limits enforced
\end{enumerate}

\section{Usability Testing}

Usability testing was conducted internally by the project team members to evaluate how intuitive and accessible the platform is. By performing a series of structured tasks using the application, the team members systematically identified areas for improvement and validated the overall design of the user interface. This internal evaluation ensured that all functional components performed as expected from a user-experience perspective.
Based on this internal feedback, several UI/UX refinements were implemented, including the addition of tooltips for all toolbar buttons, larger text options for improved readability, and progress indicators for asynchronous AI operations. Furthermore, error messages were enhanced with clearer action steps, and loading states were introduced to provide better visual feedback during data processing tasks.

Common bugs resolved:
\begin{enumerate}
    \item Edge IDs missing in AI response (added automatic generation)
    \item JSON parsing errors with unescaped newlines (added regex extraction)
    \item Undo/Redo not working for edge edits (added edge state to snapshots)
    \item Auto-layout misaligning decision nodes (adjusted width calculation)
    \item Theme preference not persisting (added localStorage persistence)
    \item Node resize handles invisible (added CSS styling)
    \item Edge labels not saving (added updateEdgeLabel function)
    \item Image upload timeout for large files (added thumbnail resizing)
    \item CORS errors in production (configured allowed origins properly)
\end{enumerate}

\section{Conclusion}

Automated Diagram Digitization has been thoroughly tested and meets all functional requirements. With a 98.6\% test pass rate, 95.3\% bug resolution, and positive user feedback (4.3/5 satisfaction), the system is ready for deployment. The remaining two low-priority bugs are documented for future development sprints. Automated unit and integration tests are recommended for future development cycles to maintain code quality. Continuous testing and monitoring will help ensure long-term stability and reliability as new features are introduced. These practices will also support efficient maintenance and scalable development of the platform in subsequent versions.
