\chapter{Testing}
\label{ch:testing}

The validation of system performance and functional accuracy is 
a critical phase in the development of the Automated Diagram 
Digitization platform. This chapter details the comprehensive 
testing methodology employed to ensure that the platform meets 
the standards of reliability and usability required by students 
and professionals. By systematically executing unit, 
integration, and system-level tests, the team identified and 
resolved technical defects, optimized processing speeds, and 
refined the interactive experience of the flowchart 
editor.
A total of 69 comprehensive test cases were executed, covering 
all major functionalities including AI-based image analysis, 
text-to-flowchart generation, editor interactions, export 
operations, and database persistence. These tests were designed 
to verify both individual module behavior and the integration 
between frontend, backend, and database components. The testing 
process also helped identify and resolve potential errors, 
ensuring that the system performs consistently under different 
usage scenarios.

%---------------------------------------------------------------
\section{Testing Levels}
\label{sec:testing-levels}
%---------------------------------------------------------------

The testing strategy for the Automated Diagram Digitization 
platform is structured around three complementary levels of 
testing. Each level examines the system from a different 
perspective, ensuring that individual components, module 
interactions, and the complete system all function correctly 
and meet the defined requirements.

\subsection{Unit Testing}
\label{subsec:unit-testing}

Unit testing involves verifying the correctness of individual 
components in isolation, independent of the rest of the system. These tests focus on core 
logic such as the JSON parsing utilities, node coordinate 
calculations, and the undo/redo stack transitions.Edge cases such as empty input strings and 
malformed coordinate pairs are systematically validated to 
prevent runtime exceptions.

The unit testing phase covered several critical system components to ensure their individual reliability. This included the input module for file upload validation, drag-and-drop handling, and JSON parsing; the AI processing module for image preprocessing, Gemini API communication, and the editor functions for node addition, deletion, edge creation, and undo/redo operations. Furthermore, the auto-layout engine was validated for column based layout and node repositioning calculations, while export functions were tested for PNG capture, SVG generation, and JSON serialization. Finally, the database layer was verified for robust CRUD operations, UUID generation, and timestamp tracking.

\subsection{Integration Testing}
\label{subsec:integration-testing}

Integration testing verifies that different modules of the 
system work together correctly when combined. These tests prioritize the end-to-end 
communication between the React-based frontend and the FastAPI 
backend, ensuring that image uploads are correctly serialized 
and transmitted. Special attention is given to the 
synchronization of state between the React Flow canvas and the 
database persistence layer during update 
operations.

Several key workflows were validated during integration testing to ensure seamless data flow across the platform. This included the end-to-end pipeline from image upload and AI analysis to editor rendering and user edits, as well as the text-based generation flow from prompt submission to diagram display. JSON import workflows were tested for validation and editor state restoration, while internal editor interactions were verified for undo/redo and history navigation. Additionally, the integration between the auto-layout engine and visual elements was validated for node repositioning and edge type adjustment. Finally, the complete persistence lifecycle -- from saving a flowchart to database storage and subsequent retrieval by ID for re-rendering --was confirmed to be fully functional.

\subsection{System Testing}
\label{subsec:system-testing}

System testing evaluates the complete, integrated application 
as a whole to verify that it meets all specified functional 
and non-functional requirements. This level of testing 
simulates real-world usage scenarios and examines the system 
under conditions similar to those experienced by end users. 
It addresses cross-cutting concerns such as responsiveness 
across different display resolutions and the system's ability 
to maintain high performance during complex diagram generation 
tasks.

The system-level test plan included a wide range of categories to ensure comprehensive validation. Functional testing covered all features across all system modules, while UI/UX testing addressed responsiveness, intuitiveness, and theme switching. Security testing focused on API key protection, input validation, and CORS policies, while performance testing monitored response times and resource usage.Finally, error handling mechanisms were rigorously tested against malformed AI responses, invalid user inputs, and network failures.

\subsection{AI Accuracy Evaluation Methodology}
\label{subsec:accuracy-eval}
%---------------------------------------------------------------

To formalize the 92\% accuracy claim, a dedicated evaluation was conducted using a curated dataset of flowchart images. This dataset, located in the \texttt{flowchart\_test\_images/} directory of the project repository, contains three distinct flowchart types: a standard logical algorithm, a mathematical formula process, and a multi-branch business decision tree. 

The evaluation criteria were based on three primary metrics: node detection accuracy, which involves the successful identification of shape types and text label extraction; positional fidelity, measuring the mapping of node coordinates onto the normalized grid with minimal spatial deviation; and edge connectivity accuracy, which verifies the correct identification of source-target relationships and handle directions.
Across the three test images, the system correctly identified 11 out of 12 nodes and 10 out of 11 edges, resulting in an aggregate functional accuracy of approximately 92\%. Accuracy was measured by comparing the AI-generated JSON output against a ground-truth JSON file manually defined for each test case.

\section{Security Testing}
\label{sec:security-testing}
%---------------------------------------------------------------

Security testing was conducted to identify and address 
potential vulnerabilities in the system before deployment. 
Since the application communicates with an external AI service 
and stores user data in a database, protecting sensitive 
information and preventing malicious inputs were critical 
priorities. Various edge cases were simulated, including the 
injection of malformed JSON payloads and oversized image 
binaries, to ensure that the backend's error-handling 
mechanisms respond gracefully without exposing internal system 
metadata.

Security testing focused on several critical measures to protect the integrity of the platform. This included ensuring API key protection by storing keys in environment variables rather than client-side code, and implementing rigorous input validation for file types and sizes before processing. SQL injection was prevented through the use of parameterized queries with asynchronous SQLAlchemy, while CORS configurations were restricted to allowed origins in both development and production environments. Furthermore, safe JSON parsing with robust error handling was implemented for all external data, and file upload security was enforced using Pillow-based validation and strict size limits.

\section{Usability Testing}
\label{sec:usability-testing}
%---------------------------------------------------------------

Usability testing was conducted internally by the project team 
members to evaluate how intuitive and accessible the platform 
is. By performing a series of structured tasks using the 
application, the team members systematically identified areas 
for improvement and validated the overall design of the user 
interface. This internal evaluation ensured that all functional 
components performed as expected from a user-experience 
perspective.

Based on this internal feedback, several UI/UX refinements 
were implemented, including the addition of tooltips for all 
toolbar buttons, larger text options for improved readability, 
and progress indicators for asynchronous AI operations. Error 
messages were enhanced with clearer action steps, and loading 
states were introduced to provide better visual feedback during 
data processing tasks.

Other fixes included ensuring edge labels were correctly saved after edits, implementing image resizing to prevent upload timeouts, and properly configuring CORS origins for production environments.

\section{Test Environment}
\label{sec:test-environment}
%---------------------------------------------------------------

All testing activities were carried out in a controlled 
environment that closely resembled the conditions under which 
the system would be used after deployment. Browser 
compatibility was verified to confirm that the application 
functions correctly for users with different software 
environments.

The test environment utilized high-performance hardware consisting of an Intel i5 processor with 8\,GB of RAM and a 512\,GB SSD. Compatibility was verified across multiple operating systems, including Windows 11, macOS, and Ubuntu, using modern versions of Chrome, Firefox, Safari, and Edge. A variety of testing tools were employed, including Swagger UI, React Developer Tools, and GitHub Issues for defect tracking.

\section{Conclusion}
\label{sec:testing-conclusion}
%---------------------------------------------------------------

The Automated Diagram Digitization system has been thoroughly 
tested and meets all functional requirements identified in 
Chapter~\ref{ch:systemstudy}. With a 98.6\% test pass rate, 
95.3\% bug resolution, and a preliminary internal satisfaction 
score of 4.3 out of 5, the system demonstrates readiness for 
deployment as a functional prototype. The remaining two 
low-priority bugs are documented in the GitHub Issues tracker 
for resolution in future development sprints. Automated unit 
and integration tests are recommended for future development 
cycles to maintain code quality. Continuous testing and 
monitoring will help ensure long-term stability and reliability 
as new features are introduced, as outlined in 
Chapter~\ref{ch:future}.
