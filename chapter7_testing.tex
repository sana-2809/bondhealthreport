\chapter{Testing}
\label{ch:testing}

\section{Introduction}
Testing ensures the reliability, functionality, and quality of the Automated Diagram Digitization system across all modules. Over 50 comprehensive test cases covered all major functionalities including AI-based image analysis, text-to-flowchart generation, editor interactions, export operations, and database persistence.

\section{Testing Levels}

\subsection{Unit Testing}
Tested individual components in isolation:
\begin{itemize}
    \item \textbf{Input Module}: File upload validation, drag-drop handling, JSON parsing
    \item \textbf{AI Processing}: Image preprocessing, Gemini API communication, JSON response parsing
    \item \textbf{Editor Functions}: Node addition/deletion, edge creation, undo/redo operations
    \item \textbf{Auto-Layout}: Column clustering algorithm, node repositioning calculations
    \item \textbf{Export Functions}: PNG capture, SVG generation, JSON serialization
    \item \textbf{Database}: CRUD operations, UUID generation, timestamp tracking
\end{itemize}

\subsection{Integration Testing}
Verified module interactions through key workflows:
\begin{itemize}
    \item Image upload $\rightarrow$ AI analysis $\rightarrow$ Editor rendering $\rightarrow$ User edits
    \item Text prompt $\rightarrow$ Gemini generation $\rightarrow$ JSON parsing $\rightarrow$ Diagram display
    \item JSON import $\rightarrow$ Validation $\rightarrow$ Editor state restoration
    \item Editor edits $\rightarrow$ Undo/Redo $\rightarrow$ History navigation
    \item Auto-layout $\rightarrow$ Node repositioning $\rightarrow$ Edge type adjustment
    \item Save flowchart $\rightarrow$ Database storage $\rightarrow$ Retrieve by ID $\rightarrow$ Re-render
\end{itemize}

\subsection{System Testing}
Comprehensive testing across all requirements:
\begin{itemize}
    \item \textbf{Functional testing}: All 40+ features across all modules
    \item \textbf{UI/UX testing}: Responsiveness, intuitiveness, theme switching
    \item \textbf{Security testing}: API key protection, input validation, CORS
    \item \textbf{Performance testing}: Response times and resource usage
    \item \textbf{Cross-browser testing}: Chrome, Firefox, Safari, Edge
    \item \textbf{Responsive design testing}: Desktop, tablet, mobile views
    \item \textbf{Error handling}: Malformed AI responses, invalid inputs, network failures
\end{itemize}

\section{Security Testing}

Security testing focused on protecting the Gemini API key and preventing malicious inputs:

\begin{itemize}
    \item \textbf{API Key Protection}: Key stored in environment variables, never in client code
    \item \textbf{Input Validation}: File type and size validation before processing
    \item \textbf{SQL Injection}: Parameterized queries with async SQLAlchemy
    \item \textbf{CORS Configuration}: Restricted to allowed origins (localhost:5173, localhost:3000)
    \item \textbf{JSON Parsing}: Safe parsing with error handling for malformed data
    \item \textbf{File Upload Security}: Image validation using Pillow, size limits enforced
\end{itemize}

Common bugs resolved:
\begin{itemize}
    \item Edge IDs missing in AI response (added automatic generation)
    \item JSON parsing errors with unescaped newlines (added regex extraction)
    \item Undo/Redo not working for edge edits (added edge state to snapshots)
    \item Auto-layout misaligning decision nodes (adjusted width calculation)
    \item Theme preference not persisting (added localStorage persistence)
    \item Node resize handles invisible (added CSS styling)
    \item Edge labels not saving (added updateEdgeLabel function)
    \item Image upload timeout for large files (added thumbnail resizing)
    \item CORS errors in production (configured allowed origins properly)
\end{itemize}
