\chapter{Implementation}
\label{ch:implementation}

This chapter details the implementation of the Automated Diagram Digitization system, describing each module's logic, API interactions, and integration with the broader application. The implementation follows a layered architecture: FastAPI handles HTTP routing, PostgreSQL serves as the persistent store, and React with Vite presents the user interface. All code is organised within separate frontend and backend directories, with environment-specific settings separated into a \texttt{.env} file.

% ─────────────────────────────────────────────────────────────────────────────
\section{Development Environment Setup}
\label{sec:dev-env}

The backend runs on Python 3.9+ with the FastAPI framework. The database is a PostgreSQL instance with JSONB support for flexible schema storage. The frontend runs on Node.js v18+ with React 18 and Vite for fast development. Sensitive configuration values — database connection strings and Gemini API keys — are stored in a \texttt{.env} file at the project root and loaded via the \texttt{python-dotenv} package.

The development workflow was organized using separate frontend and backend directories to maintain modularity. The frontend was developed using Node.js and npm for dependency management, while the backend used a Python virtual environment to isolate project libraries. This separation ensures that dependencies remain controlled and reproducible across different development machines.

Version control was maintained using Git, with the project repository hosted on GitHub. This enabled structured version tracking, collaborative development, and integration with deployment platforms. Source code changes were tested locally before being committed to the repository, ensuring stability of the application during development.

\section{Backend Implementation}
The backend of the Automated Diagram Digitization platform is responsible for handling all core processing tasks, including AI-based diagram recognition, data validation, database interaction, and communication with the frontend interface. Implemented using the FastAPI framework, the backend exposes a set of RESTful endpoints that allow the frontend application to submit images or text prompts and receive structured flowchart data in JSON format.

The backend architecture emphasizes modularity and reliability. Each endpoint performs a clearly defined task, while shared utilities handle operations such as image preprocessing, prompt management, and response validation. This design simplifies maintenance and allows the backend to scale efficiently as additional features are introduced.

\label{sec:backend}

\subsection{Image Analysis Endpoint}

The image analysis endpoint accepts an uploaded flowchart image, processes it, sends it to Gemini AI with a custom system prompt, and returns structured JSON containing nodes and edges. The image is resized to a thumbnail of 1024x1024 pixels for optimal API performance while maintaining aspect ratio.



\subsection{Text-to-Flowchart Endpoint}

The text-to-flowchart endpoint accepts a natural language description of an algorithm and returns a generated flowchart. The user prompt is combined with a specialized system prompt designed for flowchart generation. The response is cleaned using regex extraction to handle any explanatory text the AI might add, ensuring only valid JSON is returned.

\subsection{Database Operations}

The database layer uses SQLAlchemy with async support for PostgreSQL operations. The flowcharts table stores each diagram with a UUID primary key, title, JSONB column for the complete React Flow state, and timestamps for creation and updates. The JSONB column provides flexibility to store varying node and edge structures without requiring schema changes.

\section{Frontend Implementation}
The frontend of the Automated Diagram Digitization system provides the interactive environment through which users create, edit, and export diagrams. Built using React with Vite, the frontend focuses on delivering a responsive and intuitive user experience while maintaining high performance for interactive diagram editing.

React Flow is used as the primary diagram rendering library, enabling dynamic visualization of nodes and edges within the flowchart editor. Additional interface components such as sidebars, toolbars, and input forms are implemented using reusable React components. This component-based structure simplifies interface maintenance and allows the system to be extended with additional features in the future.

\label{sec:frontend}

\subsection{Input Page}

The input page provides three methods for creating flowcharts: image upload with drag-drop support, text prompt for algorithm description, and JSON import for existing flow data. A tabbed interface allows users to switch between input methods, with each tab showing the appropriate form controls.

\subsection{Undo/Redo Functionality}

Undo and redo operations are implemented using keyboard controls to allow quick correction of editing actions within the flowchart editor. The system listens for standard keyboard shortcuts and updates the diagram state accordingly. Pressing \texttt{Ctrl+Z} performs an undo operation, restoring the diagram to its previous state, while \texttt{Ctrl+Y} or \texttt{Ctrl+Shift+Z} performs a redo operation to reinstate the most recently undone action.

The implementation relies on maintaining a history of diagram states. Whenever a modification occurs—such as node movement, edge creation, deletion, or label editing—the previous state of the nodes and edges is stored. During an undo operation, the editor restores the most recent saved state, ensuring that accidental changes can be reversed quickly. This approach improves usability and enables users to experiment with diagram modifications without the risk of permanently losing previous configurations.

\subsection{Custom Node Components}

Each flowchart shape is implemented as a custom node component extending a common base node. The base node provides shared functionality such as connection handles on all four sides, double-click text editing, and node resizing capabilities. This modular structure ensures consistent behavior across all node types while allowing shape-specific styling.

Different visual styles are applied using CSS. Process nodes are rendered as rectangles, decision nodes as diamonds using a \texttt{clip-path} polygon, start/end nodes as rounded shapes, and input/output nodes as parallelograms. This approach enables the editor to represent standard flowchart symbols while keeping the code modular and maintainable.


\subsection{Auto-Layout Feature}

The auto-layout feature uses a column-based clustering algorithm to align nodes neatly. It calculates the true center X-coordinate for each node, clusters nodes within a tolerance of 150 pixels, computes the average center for each cluster, and repositions nodes to align with their cluster's average. Edge types are automatically adjusted based on alignment — straight edges for vertically aligned nodes, smoothstep edges for others. This automatic arrangement improves the readability of complex diagrams by reducing overlaps and uneven spacing between nodes. As a result, users can quickly organize large flowcharts into a clean and structured layout with minimal manual adjustment.

Pressing Enter saves the label, Escape cancels, and clicking elsewhere also saves. Labels are displayed with a background box for readability, with styling that adapts to the current theme.

\subsection{Node Colouring and Text Styling}

The flowchart editor provides styling options that allow users to customize the visual appearance of nodes and text. Users can change the background color of nodes using a color picker interface, making it easier to visually distinguish different stages or categories within a flowchart. This feature is particularly useful for highlighting important steps, grouping related processes, or improving the overall readability of complex diagrams.

In addition to node coloring, the editor supports text styling options including bold, italic, and adjustable font size. These controls allow users to emphasize key information within nodes and improve the clarity of diagram content. By combining node color customization with flexible text formatting, the editor enables users to create visually structured and easily interpretable flowcharts.

\subsection{Theme Implementation}

The system supports both dark and light modes using CSS variables. A theme context provides the current theme state and toggle function. User preference is persisted in localStorage so that the selected theme remains consistent across sessions. CSS variables control node backgrounds, borders, text colors, canvas background, and grid colors, with all components reacting to theme changes. This approach ensures a centralized and maintainable styling system across the application. The dark mode helps reduce eye strain during prolonged use, especially in low-light environments, while the light mode provides a clear interface suitable for documentation and presentations. Theme switching is performed instantly without reloading the page, ensuring a smooth user experience.

\subsection{Export Functionality}

Three export formats are supported: PNG for sharing, SVG for vector graphics, and JSON for data portability. PNG export uses the html-to-image library to capture the canvas as a high-resolution image suitable for documents and presentations. SVG export uses React Flow’s built-in rendering to generate scalable graphics that maintain clarity at any resolution. JSON export stores the nodes and edges structure with formatted output, allowing diagrams to be saved, transferred, and re-imported for further editing without losing layout information.

Providing multiple export formats ensures that the generated diagrams can be used across different environments and workflows. PNG images are suitable for documentation and presentations, SVG files allow further editing in vector graphic tools, and JSON exports enable users to reload and continue editing diagrams later. This flexibility ensures that the system supports both visual sharing and structured data storage.

\section{Summary}
\label{sec:implementation-summary}

This chapter presented the implementation of modules within the Automated Diagram Digitization platform. From the AI-powered image analysis endpoint to the React Flow editor, each component was built with emphasis on accuracy, user experience, and maintainability. The layered architecture ensures modules remain testable and extensible. The implementation integrates multiple technologies including FastAPI, React, PostgreSQL, and the Gemini AI API to provide a complete workflow from diagram input to final export. Each module was designed to interact seamlessly with others while maintaining clear separation of responsibilities. This modular structure simplifies future enhancements and maintenance of the system. The following chapter describes the testing strategy used to validate the correctness and robustness of the implementation.
